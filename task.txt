Задание

У нас есть API https://reqres.in/, для нее надо написать интерфейс клиента
Функциональность:
- сервис не доступен без авторизации, на все клиентские url показывать окно авторизации
- возможность регистрации, с возможностью открывать этот роут по url, после успешной регистрации кидает на страницу авторизации 
- авторизованному клиенту доступны все страницы кроме регистрации
- после авторизации клиен видит главную страницу со списком пользователей
- роутинг на стороне клиента

Страницы: 
- авторизации. С формой авторизации и возможностью перехода на регистрацию
- регистрации. С формой регистрации
- главная. шапка с кнопками выхода и просомтра данных текущего авторизованного пользователя, список пользователей в виде иконок
- просмотр пользователя. шапка как на главной странице, просмотр данных конкретного пользователя в виде таблицы.
- страница 404. шапка и текст страницы 'такой страницы не существует'.

Тех. требования: 
	ReactJs, для состояния использовать redux, для роутинга react router, для верстки возможно использовать bootstrap или подобные css фреймфорки

	дизайн не критичен

	Результат отправить ввиде архива или можно залить во внешний гит репозиторий
